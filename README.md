# snhu-cs-340-client-server-development
Portfolio for my SNHU course

Writing programs that are readable, maintainable, and adaptable in my experience has been about writing clean modular code. Separating the what from the how, in other words, hiding implementation details from higher level functions, makes it easier to make changes to parts of the code that need it without having to touch everything thta works. The approach of this project was a good example by creating a complete CRUD module and then utilizing it in the final web dashboard. The CRUD module can then be reused in other projects, but also, the implementation of that module can be changed to accomodate different database technologies without having to change the dashboard code, so long as the API does not change.

Developing a software solution for a client should always about delivering a product that meets a set of requirements. Sometimes those requirements may not be well communicated. It is the job of the developer to attemot to understand their client's needs so that they may be fully addressed even if they are not well communicated. This project delivered a workable dashboard based on initial requirements, but gaining client feedback and working with the customer to enhance UX, based on budget, would improve the product overall. In any service industry, especially a skilled service, it is your job to use your skills to provide the best overall result for your client.
